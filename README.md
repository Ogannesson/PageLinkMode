# PageLinkMode

PageLinkMode 是一个面向 Chrome 的 Manifest V3 扩展，用来统一接管网页内部的链接跳转行为。

它的目标不是拦截浏览器所有“开新标签页”的动作，而是聚焦在你真正能感知到、也最常需要调整的网页内导航场景，例如：

- 论坛帖子列表点击帖子
- 视频站点点击视频卡片
- 页面脚本通过 `window.open(...)` 打开新页
- 常见 HTML 表单提交跳转

你可以把它理解为一个“网页内跳转策略控制器”：

- 默认让网页链接在当前标签页打开
- 或默认让网页链接在新标签页打开
- 再针对单个站点或单个具体页面设置例外规则

## 特性

- 支持全局默认模式切换：
  - `同标签页打开`
  - `新标签页打开`
- 支持三层优先级规则：
  - 页面规则
  - 站点规则
  - 全局默认规则
- 支持在 popup 中快速设置：
  - 当前站点规则
  - 当前页面规则
  - 全局默认模式
- 支持 `window.open(...)` 接管
- 支持普通链接点击接管
- 支持常见 HTML 表单提交接管
- 支持规则持久化保存
- 提供独立的规则管理页，统一查看和删除站点规则、页面规则

## 规则优先级

PageLinkMode 的规则按照以下顺序生效：

1. 页面规则
2. 站点规则
3. 全局默认规则

也就是说：

- 如果某个具体页面配置了规则，就优先使用该页面规则
- 否则，如果当前站点配置了规则，就使用站点规则
- 如果都没有配置，就回退到全局默认模式

页面规则的键采用规范化 URL：

- 使用 `origin + pathname + search`
- 忽略 `hash`

站点规则的键采用精确 `hostname`。

## 工作原理

扩展主要采用“事前拦截”的方式处理网页导航，而不是等新标签已经打开后再强行关闭。

当前实现由以下几个部分组成：

- `service worker`
  - 负责读取和写入规则
  - 负责计算当前页面实际生效的模式
  - 负责统一执行 `tabs.update` / `tabs.create`
- `content script`
  - 在 `document_start` 注入
  - 负责拦截文档级点击事件和表单提交事件
- 主世界注入脚本
  - 用于接管页面脚本内部的 `window.open(...)`
- popup
  - 用于当前页面上下文下的快速切换
- options 页面
  - 用于集中查看和管理所有规则

这种实现方式的优点是：

- 不需要等标签打开后再关闭，交互更自然
- 更适合论坛、内容站、视频站这类网页内导航
- 更容易表达“站点级”和“页面级”的规则模型

## 当前支持的行为

当前版本已经覆盖以下网页内导航行为：

- 普通 `<a href="...">` 链接点击
- `target="_blank"` 链接
- `window.open(...)`
- 常见 HTML 表单提交

为了避免破坏用户的主动操作，以下行为默认保留原始语义，不由扩展强制改写：

- `Ctrl / Cmd / Shift / Alt` 修饰键点击
- 中键点击
- 下载链接
- `mailto:` 链接
- `javascript:` 链接
- 同页锚点跳转

## 当前不支持的范围

PageLinkMode 当前明确 **不尝试覆盖** 以下场景：

- `chrome://` 页面
- 扩展页
- Chrome Web Store 页面
- 浏览器地址栏、书签栏、标签栏等原生 UI
- 浏览器右键菜单中的“在新标签页打开链接”
- 纯 `history.pushState` / `replaceState` 驱动的站内前端路由切换

这不是 bug，而是当前版本的设计边界。

## 安装

当前项目以本地开发加载为主。

### 从源码构建

```powershell
npm install
npm run build
```

构建完成后，产物位于：

```text
dist/
```

### 在 Chrome 中加载

1. 打开 `chrome://extensions`
2. 启用右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择 `C:\CodeProgram\PageLinkMode\dist`

## 使用方式

1. 打开一个普通网页
2. 点击浏览器工具栏中的 PageLinkMode 图标
3. 如果当前站点尚未授权，先点击“授权当前站点”
4. 根据需要设置：
   - 全局默认模式
   - 当前站点规则
   - 当前页面规则
5. 如果要集中查看、修改或删除已有规则，打开“规则管理”

```text

  background/   service worker
```

## 已知限制

- 某些高度定制的站点可能不会通过标准链接或标准表单进行跳转，这类页面可能仍需要继续适配
- 当前版本没有实现基于路径前缀、正则或通配符的高级规则匹配
- 当前版本没有实现针对 SPA 路由的通用接管
- 当前版本仅针对 Chrome 生态验证，尚未为 Firefox / Edge 做兼容性整理
